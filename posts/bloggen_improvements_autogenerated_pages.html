Auto-generated pages for bloggen.py and more


2015-09-27 by dvolk

Bloggen.py now has options to auto-generate pages.

This is useful for example for making an index containing links to all pages on the blog, or indexes of pages with specific titles.

Auto-generated pages

Functions are simply added to the special_pages list:

# to generate posts from functions rather than files,
# add them to special_pages. The elements of special pages are either
# function returning a post, or a tuple of a function returning a Post
# and a tuple of arguments to that function
special_pages = [all_posts_by_date,
                 (all_posts_about, ('Project X', 'project_x')),
                 (all_posts_about, ('bloggen.py', 'bloggen_py'))]

The functions must return a Post object, e.g.:

def all_posts_by_date():
    global posts
    rest = ""
    for post in posts:
        rest += "<p><a href='{0}.html'>{1}</a>".format(post.link, post.title)
        if post.time:
            time_str = datetime.datetime.strftime(post.time, post_time_output_fmt)
            rest += " <span class='date_text'>{0}</span>".format(time_str)
        rest += "</p>\n"
    return Post("all_posts_by_date", None, None, None,
                "All posts by date", rest, False)

List of all auto-generated pages

It also creates a list of all auto-generated pages.

Link replacement

Links from posts to other posts can now use __SITEPOST__ as a keyword that gets replaced with site_url/posts. Similarly __SITEROOT__ refers to site_url. This makes it possible to easily link between posts without encoding the full URL in the post.

The site_url can also be overried by a new command-line option --root
